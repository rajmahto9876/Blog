The **NVIC (Nested Vectored Interrupt Controller)** is a component of ARM Cortex-M processors that manages interrupt handling. It provides a interrupt management system with nested interrupt support, allowing real-time systems to handle multiple interrupts with different priority levels efficiently.

The NVIC is a built-in interrupt controller in all ARM Cortex-M processors (M0, M3, M4, M7, etc.) that:
- Manages external and internal interrupt sources
- Provides priority-based interrupt handling
- Supports interrupt nesting
- Allows dynamic enable/disable of interrupts
- Provides interrupt status monitoring

## NVIC Architecture

The NVIC sits between interrupt sources and the processor core, handling interrupt prioritization and delivery. It typically manages **up to 240 external interrupt sources** (depending on the specific Cortex-M variant), plus system exceptions.

```
Interrupt Sources
       ↓
    NVIC
       ↓
 Cortex-M Core
```

## Key Components

### 1. **Interrupt Sources**
- **External Interrupts**: From peripherals (GPIO, UART, SPI, etc.)
- **System Exceptions**: NMI, HardFault, SysTick, PendSV, SVCall

### 2. **Priority Levels**
- Cortex-M processors support multiple priority levels (4-256 depending on implementation)
- Lower numerical values = Higher priority
- Configurable number of priority bits

### 3. **Interrupt States**
- **Inactive**: Not pending or active
- **Pending**: Interrupt request received, waiting to be serviced
- **Active**: Interrupt handler is executing
- **Active and Pending**: Interrupt re-triggered while handler running

## NVIC Registers

### Enable/Disable Registers

**NVIC_ISER (Interrupt Set-Enable Register)**
```c
// Enable interrupt n (0-239)
NVIC_ISER[n/32] |= (1 << (n % 32));
```

**NVIC_ICER (Interrupt Clear-Enable Register)**
```c
// Disable interrupt n
NVIC_ICER[n/32] |= (1 << (n % 32));
```

### Pending Registers

**NVIC_ISPR (Interrupt Set-Pending Register)**
```c
// Manually set interrupt n as pending
NVIC_ISPR[n/32] |= (1 << (n % 32));
```

**NVIC_ICPR (Interrupt Clear-Pending Register)**
```c
// Clear pending status of interrupt n
NVIC_ICPR[n/32] |= (1 << (n % 32));
```

### Priority Registers

**NVIC_IPR (Interrupt Priority Register)**
```c
// Set priority of interrupt n
// Each interrupt gets 8 bits (upper 4 bits used on Cortex-M3/M4)
NVIC_IPR[n] = (priority << 4);  // For Cortex-M3/M4
```

### Active Bit Registers

**NVIC_IABR (Interrupt Active Bit Register)**
```c
// Read to check if interrupt n is currently active
uint8_t is_active = (NVIC_IABR[n/32] >> (n % 32)) & 1;
```

## Practical Usage Examples

### Enable an Interrupt

```c
#include "core_cm4.h"  // CMSIS header

void enable_peri_interrupt(void)
{
    // Enable Peripheral interrupt (assuming IRQ number is 36)
    NVIC_EnableIRQ(36);
}
```

### Disable an Interrupt

```c
void disable_peri_interrupt(void)
{
    // Disable Peripheral interrupt
    NVIC_DisableIRQ(36);
}
```

### Set Interrupt Priority

```c
void set_uart_priority(void)
{
    // Set priority of UART interrupt
    // 0 = highest priority, 255 = lowest (for 256-level system)
    NVIC_SetPriority(36, 5);
}
```

### Clear Pending Interrupt

```c
void clear_pending_uart(void)
{
    // Clear pending interrupt if already triggered
    NVIC_ClearPendingIRQ(36);
}
```

### Check Interrupt Status

```c
void check_interrupt_status(void)
{
    // Check if interrupt is enabled
    uint32_t enabled = NVIC_GetEnableIRQ(36);
    
    // Check if interrupt is pending
    uint32_t pending = NVIC_GetPendingIRQ(36);
    
    // Check if interrupt is active
    uint32_t active = NVIC_GetActive(36);
}
```

## Priority Configuration

### Understanding Priority Levels

In Cortex-M3/M4:
- **8-bit priority field**, but typically only upper 4 bits used
- **Configurable priority groups** for preemption
- **Lower numerical values = Higher priority**

### Priority Example

```c
// Assuming 16 priority levels (4 bits used)
NVIC_SetPriority(UART0_IRQn, 3);  // High priority
NVIC_SetPriority(GPIO_IRQn, 12);  // Low priority
```

### Priority Groups

```c
// Set priority grouping - determines preemption and subpriority bits
// Value 0: No preemption (all bits for subpriority)
// Value 3: 8 preemption levels, 2 subpriority levels
NVIC_SetPriorityGrouping(3);
```

## Interrupt Nesting

The NVIC supports **nested interrupts**, where a higher-priority interrupt can preempt a lower-priority interrupt handler.

```c
// Handler for high-priority interrupt
void TIMER_IRQHandler(void)
{
    // This can be interrupted by higher priority interrupt
}

// Handler for lower-priority interrupt  
void GPIO_IRQHandler(void)
{
    // This will be preempted if higher priority interrupt occurs
}
```

## Exception vs Interrupt

### System Exceptions
- **NMI**: Non-Maskable Interrupt (highest priority, cannot be disabled)
- **HardFault**: Hardware error trap
- **MemManage**: Memory protection fault
- **BusFault**: Bus error
- **UsageFault**: Instruction usage error
- **SVCall**: Supervisor call
- **PendSV**: Pendable service call
- **SysTick**: System tick timer

### External Interrupts
- Typically 16-240 depending on MCU variant
- Managed like regular interrupts

## NVIC Memory Map

Typical NVIC registers are at memory address **0xE000E100 - 0xE000E300**:

- **0xE000E100**: ISER (Set-Enable)
- **0xE000E180**: ICER (Clear-Enable)
- **0xE000E200**: ISPR (Set-Pending)
- **0xE000E280**: ICPR (Clear-Pending)
- **0xE000E300**: IABR (Active Bit)
- **0xE000E400**: IPR (Priority)

## Debugging NVIC

```c
// Function to dump NVIC status
void print_nvic_status(uint8_t irq_num)
{
    uint8_t index = irq_num / 32;
    uint8_t bit = irq_num % 32;
    
    printf("IRQ %d Status:\n", irq_num);
    printf("  Enabled: %s\n", 
        (NVIC->ISER[index] >> bit) & 1 ? "YES" : "NO");
    printf("  Pending: %s\n",
        (NVIC->ISPR[index] >> bit) & 1 ? "YES" : "NO");
    printf("  Active: %s\n",
        (NVIC->IABR[index] >> bit) & 1 ? "YES" : "NO");
    printf("  Priority: %d\n",
        NVIC->IP[irq_num] >> 4);
}
```
